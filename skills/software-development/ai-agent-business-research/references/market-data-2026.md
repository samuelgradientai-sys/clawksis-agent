## Key Security Code Review Gaps (Opportunities)

1. **Framework-specific review** — Rails, Django, Next.js idioms (whitespace exists per major framework)
2. **Security-first PR review** — buyer is AppSec team, not engineering (empty space)
3. **Legacy/untested codebases** — Java 8, COBOL, PHP 5 (large enterprises)
4. **Review for AI-generated PRs** — when Claude Code/Codex opens a PR (totally new)
5. **Review observability** — dashboards for engineering managers (early)
6. **Multi-model review** — no product does consensus across multiple LLMs for security findings
